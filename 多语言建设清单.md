# AI Beauty Stylist 多语言建设清单

> 目标：把多语言版本当成新的 SEO 增长资产，而不是把英文站逐字翻译一遍。  
> 适用范围：首页、Try-On 工具页、Shade Finder、Undertone Quiz、Foundation Shade Finder、Makeup Guides、Style Guides、Scenario Guides、Lipstick Shade Guides、Legal 页面、sitemap、hreflang、Search Console。  
> 当前判断：本项目已经具备英文 SEO 内容集群和 sitemap/audit 基础，多语言的下一步不是先翻全部页面，而是选择高意图语言和高商业意图页面做可索引试点。

---

## 1. 从 SunriseTime 清单可直接借鉴的原则

### 1.1 多语言是 SEO 扩张，不是 UI 装饰

- [ ] 每个语言版本必须有独立 URL。
- [ ] 每个语言版本必须有目标语言的 title、description、H1、正文、FAQ、schema。
- [ ] 不能只翻导航、按钮、页脚，页面主体仍保持英文。
- [ ] 不能用 query 参数切语言，如 `?lang=es`。
- [ ] 不能靠 cookie 或浏览器语言切换同一个可索引 URL 的内容。
- [ ] 未达到完整页面质量的语言，不应上线收录。

### 1.2 本项目特别要吸收的一点

SunriseTime 清单最重要的启发是：

> 不同语言不是英文关键词的机械翻译，而是新的关键词研究和搜索意图重写。

对 AI Beauty Stylist 来说尤其重要，因为美妆关键词在不同语言里差异很大：

- 英文用户搜 `AI makeup try on`。
- 西语用户可能搜 `probador de maquillaje virtual` / `maquillaje virtual online`。
- 葡语用户可能搜 `simulador de maquiagem` / `testador de maquiagem virtual`。
- 日语用户会用 `ブルベ` / `イエベ` / `似合うリップ` 等本地表达。
- 韩语用户会用 `웜톤` / `쿨톤` / `퍼스널 컬러`。

所以多语言内容不能只做翻译，必须按语言重写 SEO 标题、首屏答案和 FAQ。

## 3. 推荐语言优先级

### 3.1 第一阶段试点语言

| 语言 | 代码 | 推荐优先级 | 原因 |
|---|---:|---:|---|
| 简体中文 | `zh-CN` | P0 | 用户明确要求进入 P0；中文美妆搜索中“AI 试妆、适合我的口红、粉底色号、冷暖皮判断”具备强工具型意图，适合先用 `/zh/` 做可索引试点。 |
| 西班牙语 | `es` | P0 | 美妆、AI tool、shade finder 搜索空间大；拉美市场大；内容成本可控。 |
| 葡萄牙语 | `pt` | P0 | 巴西美妆需求强，`maquiagem` 相关搜索意图商业价值高。 |
| 法语 | `fr` | P1 | 欧洲/加拿大市场，beauty guide 和 skincare/makeup 内容成熟。 |
| 日语 | `ja` | P1 | `イエベ/ブルベ`、personal color、lip shade 搜索表达成熟，适合 shade/undertone 工具。 |
| 韩语 | `ko` | P1 | K-beauty 语境强，适合 `korean dewy makeup`、personal color、tone matching。 |

### 3.2 第二阶段语言

| 语言 | 代码 | 推荐优先级 | 原因 |
|---|---:|---:|---|
| 德语 | `de` | P2 | 工具类 SEO 可做，但美妆长尾需先验证。 |
| 印尼语 | `id` | P2 | 东南亚美妆市场大，后续可结合 halal/skin tone/Asian skin 内容。 |
| 阿拉伯语 | `ar` | P3 | 市场有价值，但 RTL、文化适配、图像审美、文案审核成本更高。 |
| 繁体中文 | `zh-Hant` | P3 | 可做香港/台湾美妆内容，但需要独立关键词研究，不建议简单用简中直转。 |

### 3.3 不建议第一阶段做的语言

- [ ] 不建议一开始同时铺 8-10 种语言。
- [ ] 不建议先做阿语，除非准备好 RTL、文化审美和本地化审核。
- [ ] 简体中文已按当前产品策略提升为 P0，但必须用中文真实搜索表达重写，不得把英文页面机械翻译后上线。

## 4. URL 路由策略

### 4.1 推荐第一阶段方案

英文保留根路径：

```text
/
/try-on/
/tools/shade-finder/
/tools/undertone-quiz/
/tools/foundation-shade-finder-by-undertone/
/styles/monolid-makeup/
/scenarios/wedding-guest-makeup/
/lipstick-shades/asian-skin-lipstick-shades/
```

其他语言使用语言前缀，但第一阶段不翻译核心路由段：

```text
/zh/
/zh/try-on/
/zh/tools/shade-finder/
/zh/tools/undertone-quiz/
/zh/tools/foundation-shade-finder-by-undertone/
/zh/styles/monolid-makeup/
/zh/scenarios/wedding-guest-makeup/
/zh/lipstick-shades/asian-skin-lipstick-shades/
/es/
/es/try-on/
/es/tools/shade-finder/
/es/tools/undertone-quiz/
/es/tools/foundation-shade-finder-by-undertone/
/es/styles/monolid-makeup/
/es/scenarios/wedding-guest-makeup/
/es/lipstick-shades/asian-skin-lipstick-shades/
```

原因：

- [ ] 路由映射简单。
- [ ] hreflang 集群容易稳定。
- [ ] sitemap/audit 容易扩展。
- [ ] 避免一开始就陷入 slug 翻译、重定向、canonical 混乱。

### 4.2 第二阶段可考虑本地化 slug

当某语言已经有稳定 impressions 和索引后，再考虑 SEO slug 本地化。

示例：

```text
/zh/ai-makeup-try-on/
/zh/foundation-shade-finder/
/es/probador-maquillaje/
/pt/simulador-de-maquiagem/
/ja/personal-color-lipstick/
/ko/personal-color-makeup/
```

前提：

- [ ] 必须有 301 / canonical / hreflang 迁移方案。
- [ ] 不得让同一语言同一意图出现两个可索引 URL。
- [ ] 不得让本地化 slug 和英文 slug 同时收录并互相竞争。

### 4.3 禁止项

- [ ] 禁止 `/try-on?lang=es` 作为可索引语言页。
- [ ] 禁止根据浏览器语言自动把 `/try-on/` 改成西语内容。
- [ ] 禁止多个语言共用同一 canonical。
- [ ] 禁止西语页 canonical 到英文页。

## 6. 页面级 SEO 标准

每个多语言可索引页面必须满足以下标准。

### 6.1 Title

- [ ] 必须用目标语言写。
- [ ] 必须包含该语言真实主关键词。
- [ ] 不能直接从英文 title 逐字翻译。
- [ ] 品牌词可以保留 `AI Beauty Stylist`。

示例方向：

```text
EN: Free AI Lipstick and Foundation Shade Finder
ES: Probador de tonos de labial y base con IA
PT: Simulador de tons de batom e base com IA
ZH-CN: AI 口红和粉底色号测试
JA: AIで似合うリップとファンデーションの色を診断
KO: AI 립스틱·파운데이션 컬러 진단
```

### 6.2 Meta Description

- [ ] 必须使用目标语言。
- [ ] 必须概括页面能解决的问题。
- [ ] 必须自然出现主关键词或变体。
- [ ] 不得沿用英文 description。

### 6.3 H1

- [ ] 每页唯一。
- [ ] 必须是目标语言。
- [ ] 必须与 title 同一搜索意图。
- [ ] 不应为了翻译完整而牺牲自然表达。

### 6.4 首屏内容

工具页首屏必须先回答用户问题：

- [ ] Try-On 页：说明可以上传自拍、预览妆容、获取步骤。
- [ ] Shade Finder 页：说明可以找 lipstick/foundation shade。
- [ ] Undertone Quiz 页：说明可以判断 warm/cool/neutral/olive。
- [ ] Foundation 页：说明按 undertone 找 foundation direction。

### 6.5 正文前 100 字

- [ ] 必须出现目标语言主关键词或自然变体。
- [ ] 必须先讲用户能得到什么，再讲产品原理。
- [ ] 不得出现英文硬编码段落。

### 6.6 FAQ

- [ ] FAQ 必须按目标语言真实搜索问题重写。
- [ ] 不能逐字翻译英文 FAQ。
- [ ] FAQPage JSON-LD 必须同步目标语言。

## 8. 数据与内容架构改造清单

### 8.1 Locale 类型扩展

当前：

```ts
export type Locale = 'en';
```

目标：

```ts
export type Locale = 'en' | 'es' | 'pt' | 'fr' | 'ja' | 'ko';
```

验收：

- [ ] `getSupportedLocales()` 返回真实支持语言。
- [ ] `getLocaleFromUrl()` 能从 `/{locale}/` 解析语言。
- [ ] `localePath()` 能生成语言前缀路径。
- [ ] 英文默认不加前缀。
- [ ] 非英文统一加前缀。

### 8.2 语言包结构

建议：

```text
src/locales/en.json
src/locales/es.json
src/locales/pt.json
src/locales/fr.json
src/locales/ja.json
src/locales/ko.json
```

语言包至少覆盖：

- [ ] Header nav
- [ ] Footer
- [ ] CTA 文案
- [ ] Form labels
- [ ] Tool UI 文案
- [ ] Error/loading/success 状态
- [ ] Membership upsell 文案
- [ ] Dashboard 非 SEO UI 文案可后置

### 8.3 SEO 内容数据结构

当前英文数据主要在：

```text
src/data/stylePages.ts
src/data/scenarioPages.ts
src/data/lipstickShadePages.ts
src/data/lookCatalog.ts
```

建议不要简单把这些数组复制 5 份。推荐两层结构：

```ts
slug: 'monolid-makeup'
translations: {
  en: {...},
  es: {...},
  pt: {...}
}
```

或按语言拆文件：

```text
src/data/localized/stylePages.en.ts
src/data/localized/stylePages.es.ts
src/data/localized/scenarioPages.en.ts
src/data/localized/scenarioPages.es.ts
src/data/localized/lipstickShadePages.en.ts
src/data/localized/lipstickShadePages.es.ts
```

第一阶段建议：

- [ ] SEO 内容数据按语言拆文件。
- [ ] slug 保持英文稳定。
- [ ] 每个语言文件只放已上线页面。
- [ ] `getStaticPaths()` 只生成真实存在的语言页面。

## 9. BaseLayout 多语言改造清单

### 9.1 Props 扩展

`BaseLayout.astro` 建议支持：

- [ ] `locale`
- [ ] `canonicalPath`
- [ ] `alternateUrls`
- [ ] `dir`
- [ ] `localizedSiteName`

### 9.2 HTML lang

当前：

```html
<html lang="en">
```

目标：

```html
<html lang={locale} dir={locale === 'ar' ? 'rtl' : 'ltr'}>
```

第一阶段如果不做阿语，可以先只处理 `lang`。

### 9.3 Hreflang

每个页面必须输出真实存在的语言版本：

```html
<link rel="alternate" hreflang="en" href="https://aibeautystylist.com/tools/shade-finder/" />
```

验收：

- [ ] 每个语言版本 self-canonical。
- [ ] 每个语言版本包含自身 hreflang。
- [ ] 同一页面族的 hreflang 集合完全一致。
- [ ] 缺失语言页不得伪造 hreflang URL。

## 10. Sitemap 与 robots 标准

### 10.1 Sitemap 架构

当前已使用：

```text
/sitemap-index.xml
/sitemap-0.xml
```

多语言上线后建议升级为按语言或按页面类型拆分：

```text
/sitemap-index.xml
/sitemap-en.xml
/sitemap-es.xml
/sitemap-pt.xml
/sitemap-fr.xml
/sitemap-ja.xml
/sitemap-ko.xml
```

如果 Astro sitemap 插件不能方便拆分，可以先使用统一 child sitemap，但 audit 必须能识别语言目录。

### 10.2 Sitemap 规则

- [ ] 只包含 canonical URL。
- [ ] 只包含可索引页面。
- [ ] 不包含 dashboard/admin/login/reset/api。
- [ ] 不包含不存在语言版本的 URL。
- [ ] 不包含 redirect URL。
- [ ] 不包含 noindex URL。

### 10.3 Robots

当前 robots 指向：

```text
Sitemap: https://aibeautystylist.com/sitemap-index.xml
```

多语言上线后继续保持。

## 12. 内链与语言切换器标准

### 12.1 同语言内链优先

- [ ] `/es/tools/shade-finder/` 优先链接 `/es/lipstick-shades/.../`。
- [ ] `/pt/styles/monolid-makeup/` 优先链接 `/pt/tools/shade-finder/`。
- [ ] 不要让非英文页面大量链接回英文页面。

### 12.2 语言切换器

语言切换器必须：

- [ ] 输出真实 `<a href>`。
- [ ] 不依赖 JS 动态跳转。
- [ ] 当前页面有对应语言版本时，链接到对应语言版本。
- [ ] 当前页面没有对应语言版本时，链接到该语言首页或同主题 hub。
- [ ] 不链接到 404。

### 12.3 Makeup Guides Hub 的作用

`/makeup-guides/` 是未来多语言 SEO 分发核心。每个语言版本都应该有：

- [ ] High-intent makeup guides
- [ ] By Scenario
- [ ] By Style
- [ ] By Lip Color

## 13. 内容质量与本地化标准

### 13.1 允许使用 AI，但不能直接发布 AI 翻译

- [ ] 可以用 AI 初稿。
- [ ] 必须人工审校 SEO 页面首屏、title、description、FAQ。
- [ ] 高商业意图页面必须经过母语或高水平语言审校。

### 13.2 Beauty 内容需要额外审校

美妆内容比日出时间更容易出错，必须检查：

- [ ] 肤色表达是否自然、尊重、无冒犯。
- [ ] `Asian skin`、`deep skin`、`olive skin` 等表达在目标语言中是否合适。
- [ ] 产品建议是否与当地常见美妆语境一致。
- [ ] 日语/韩语 personal color 术语不能硬翻英文 undertone。
- [ ] 深肤色内容不能只把浅肤色建议加深一号。

### 13.3 不允许上线的内容

- [ ] 只有 title/H1 翻译，正文是英文。
- [ ] FAQ 是英文。
- [ ] JSON-LD 是英文。
- [ ] 内链锚文本是英文。
- [ ] 同一个语言页面混杂大量英文解释。
- [ ] 逐字翻译导致关键词没人搜。

## 19. 最终结论

AI Beauty Stylist 的多语言建设应遵循三条主线：

1. **先 SEO，后 UI 完美。** 先让高意图页面以目标语言可抓取、可索引、可排名。
2. **先小范围试点，后规模化。** 推荐先做 `es` 或 `pt`，验证 GSC 数据后再扩语言。
3. **先本地关键词重写，后翻译。** 美妆内容必须贴合当地搜索表达，尤其是 undertone、personal color、skin tone、lip shade、virtual try-on 等词。

只要按这个清单推进，本项目的多语言不是简单“翻译站点”，而是新的自然流量增长层。
