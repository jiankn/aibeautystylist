## 本轮已修复

### 1. smoke 测试支持所有语言

问题：

- 原 smoke 脚本依赖中文导航、中文 CTA、固定筛选词和旧版尺寸断言。
- 测试无法区分真实产品问题与测试脚本过期。

修复：

- 根据 `BASE_URL` 自动识别语言并加载对应 locale JSON。
- 导航、登录回跳、首页、试妆工作台和定价页断言改为当前语言或结构断言。
- 更新移动端页脚、场景卡片、暗黑模式和上传取消/重试断言。
- CJK 副文案长度使用适合中日韩文本的信息密度阈值。

结果：

- 英文、简中、德语、法语、日语、韩语、繁中、西语、葡语完整 smoke 全部通过。

### 2. 多语言定价路由冲突

问题：

- `/de/pricing`、`/fr/pricing`、`/ja/pricing`、`/ko/pricing`、`/zh-tw/pricing` 被 SEO 内容页占用。
- 用户从导航进入“价格/套餐”后无法看到真实套餐、价格和订阅操作。

修复：

- 将产品级 `/pricing` 设为本地化应用路由优先项。
- 保留现有本地化 SEO 标题、描述、H1 和首屏表达。
- 所有语言的 `/[locale]/pricing` 均可进入真实定价和订阅页面。

结果：

- 德语、法语、日语、韩语和繁中 smoke 中的定价页及暗黑模式检查通过。
- sitemap、canonical 和 hreflang 索引审计无错误或警告。

### 3. 动态上传和生成状态语言混用

问题：

- `public/app.js` 只识别英文和简中。
- 德语、法语、日语、韩语、繁中、西语和葡语在上传、生成、取消、失败、删除等动态状态中会回退英文或简中。

修复：

- `BaseLayout` 将当前语言已有的试妆、导航和工作台文案注入客户端。
- 动态上传、生成、取消、失败、删除、菜单和额度状态优先使用当前语言。
- 根英文页面与 API 使用同一 locale cookie，避免页面市场画像和生成结果市场画像不一致。

结果：

- 所有语言的上传、轮询、取消、重试、删除原图和删除结果路径通过 smoke。

### 4. 发现页多语言筛选错误

问题：

- 客户端把“全部”固定识别为英文 `All`。
- 韩语、繁中等页面点击场景筛选后会错误叠加默认条件，结果变成 0。

修复：

- 每个筛选组使用自身明确标记的“全部”选项，不再依赖语言文本比较。
- 筛选结果数量、摘要和清空操作使用页面当前语言标签。

结果：

- 韩语、繁中、德语、法语、日语等页面点击场景后均返回有效结果，清空后恢复 44 个妆容。

### 5. 德语、法语、日语发现页英文回退

问题：

- 德语、法语和日语发现页此前使用英文筛选、场景、质感、难度、妆容标题和顾问建议。
- 市场灵感选项也普遍回退英文。

修复：

- 新增德语、法语和日语妆容目录本地化。
- 本地化筛选、妆容标题、场景、质感、难度、卡片动作、顾问建议和注意事项。
- 补充德语、法语、日语、韩语、繁中、西语和葡语市场灵感标签。

结果：

- 德语、法语和日语发现页不再整页回退英文。
- 三种语言的 44 个妆容、筛选和顾问卡片通过 smoke。

### 6. 移动端页脚布局

问题：

- 移动端最后一列强制跨两列，造成大面积无意义留白和过高页脚。

修复：

- 移动端页脚统一使用规则的两列布局。

结果：

- 375px 宽度下九种语言无横向溢出。
- 移动端页脚高度约为 875–916px，平板约为 730–757px，桌面约为 388px。

## 上线前仍需处理

### P1：非英文定价页完整本地化

已确认：

- 非英文定价页的 URL、title、description、H1 和首屏说明已本地化。
- 套餐卡、权益对比、升级说明、FAQ、账单政策和结账弹窗仍主要使用英文。

影响：

- 不阻断查看价格和发起订阅。

建议：

2. 优先翻译套餐权益、额度规则、自动续费、取消、退款、FAQ 和 Stripe 结账说明。
3. 完成后重新跑九种语言定价页 smoke、移动端视觉和索引审计。
