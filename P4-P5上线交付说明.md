# P4-P5 上线交付说明

更新时间：2026-06-14

## 定义

- P4：生产账户、外部服务和真实域名上的发布验证。
- P5：原上线验收计划未定义 P5。本项目当前没有额外 P5 代码阻断项。

## P4 当前状态

仓库内准备已完成，真实生产执行需要 Cloudflare、Stripe、Google OAuth、邮件服务和 Google Search Console 的账户权限。

### 已准备

- `wrangler.jsonc` 已配置生产域名、D1、R2、KV、生产 provider 和 observability。
- 生产环境使用 `R2` 上传、Gemini 试妆和 Gemini 图片生成。
- 本地 `.dev.vars` 保持 mock provider，避免验收时产生付费调用。
- 数据库迁移、Stripe webhook、Google OAuth、邮件和健康检查代码已存在。
- 构建、自动化测试、索引审计、九语言审计和 smoke 已准备为发布门禁。

### 必须在生产账户执行

| 动作 | 完成标准 | 阻断生产发布 |
| --- | --- | --- |

## 推荐发布命令顺序

```powershell
npm.cmd run env:check
npm.cmd run check
npm.cmd test
npm.cmd run build
npm.cmd run audit:copy
npm.cmd run audit:blog
npm.cmd run audit:indexing
npm.cmd run audit:launch
npm.cmd run db:migrate:remote
npx.cmd wrangler deploy
```

部署后以正式域名重新执行：

```powershell
$env:BASE_URL='https://aibeautystylist.com'
npm.cmd run smoke
npm.cmd run audit:indexing
npm.cmd run audit:launch
```

真实 AI smoke 会产生外部调用和费用，只在生产 secrets、配额和数据清理策略确认后运行：

```powershell
$env:BASE_URL='https://aibeautystylist.com'
npm.cmd run smoke:real-ai
```

## P4 准入规则

只有以下条件全部满足，才可从“验收就绪”切换为“生产已发布”：

- 生产部署成功，正式域名无旧缓存或版本混用。
- 远程数据库迁移完成。
- 登录、上传、生成、额度、结账、取消和邮件路径通过。
- 正式域名 sitemap、canonical、hreflang 和 robots 复测通过。
- 生产错误日志无持续性 5xx。

## P5 结论

没有定义中的 P5 任务，也没有发现需要新增 P5 阶段才能处理的代码问题。后续工作属于上线后增长、真实用户验证和持续优化，不属于上线阻断。
