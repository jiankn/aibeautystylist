## 4. 不建议放入东亚核心包的资产

其余 37 张建议放入非东亚或 global-diverse 包。原因不是质量差，而是它们更适合作为全球多元视觉：Black、South Asian、Latina、White、Middle Eastern、Western mature 等呈现。它们应保留，服务英文、德语、法语、西语、葡语及 global 默认版本。

注意：不要为了凑满 44 张把明显非东亚图硬塞进东亚语言版本，否则会削弱中文、日文、韩文用户的第一感知。

## 5. 两个资产包的产品规划

### 5.1 短期使用两个包

短期只维护两个主包：

1. `east-asia`
   - 触发语言：`zh-CN`、`zh-TW`、`ja-JP`、`ko-KR`
   - 视觉：东亚成年人脸，清透、低饱和、水光、柔雾、证件照、通勤、面试、单眼皮/内双友好
   - 目标：让中日韩用户进入页面后第一眼觉得产品懂本地妆容需求

2. `global-diverse`
   - 触发语言：`en`、`de-DE`、`fr-FR`、`es-ES`、`pt-BR` 以及未知语言
   - 视觉：多元人群混排，不默认白人优先
   - 目标：保持国际站包容性，同时不误伤非东亚语言版本

### 5.2 长期扩展方向

长期可以从两个包扩展为更多市场包：

- `east-asia`
- `global-diverse`
- `latin-america`
- `western-europe`
- `south-asia`
- `africa`

但当前阶段不要同时扩太多，否则图片、文案、测试成本会爆炸。先把 `east-asia` 和 `global-diverse` 做稳定。

## 6. 推荐生产目录结构

当前项目已有：

- `public/images/look-{slug}.webp`
- `public/images/looks/{slug}--east-asia.webp`
- `src/data/makeup/assets.ts`
- `src/data/makeup/resolvePageAssets.ts`

建议生产目录保持两类：

### 妆容库资产

```text
public/images/looks/{slug}--east-asia.webp
public/images/looks/{slug}--global-diverse.webp
public/images/looks/thumbs/{slug}--east-asia.webp
public/images/looks/thumbs/{slug}--global-diverse.webp
```

短期兼容策略：

- 现有 `public/images/look-{slug}.webp` 继续作为 global fallback。
- 新增东亚图统一落到 `public/images/looks/{slug}--east-asia.webp`。
- 后续再逐步把 global 图复制/迁移成 `public/images/looks/{slug}--global-diverse.webp`。

### 页面级资产

建议新建：

```text
public/images/markets/east-asia/home/
public/images/markets/east-asia/blog/
public/images/markets/east-asia/seo/
public/images/markets/global-diverse/home/
public/images/markets/global-diverse/blog/
public/images/markets/global-diverse/seo/
```

示例：

```text
public/images/markets/east-asia/home/hero-before.webp
public/images/markets/east-asia/home/hero-after.webp
public/images/markets/east-asia/home/scenario-commute.webp
public/images/markets/east-asia/home/scenario-interview.webp
public/images/markets/east-asia/home/scenario-date.webp
public/images/markets/global-diverse/home/hero-before.webp
public/images/markets/global-diverse/home/hero-after.webp
```

原因：

- 妆容库资产需要按 slug 解析，所以放 `public/images/looks` 最清晰。
- 页面级资产不属于某一个妆容，放 `markets/{pack}/{page}` 更好维护。
- 不要把所有图片继续平铺到 `public/images` 根目录，否则后面 100+ 张图片会失控。

## 7. 数据层规划

### 7.1 妆容图

继续使用现有链路：

```text
locale / cookie / account preference
→ audienceContext.marketProfile
→ resolveLookCatalog(audienceContext)
→ marketVariantId
→ selectBestAsset()
→ image
```

要求：

- `east-asia` 语言版本的 44 个妆容都应该返回 `marketVariantId: "{slug}--east-asia"`。
- `global-diverse` 版本继续返回 global 或对应市场变体。
- 禁止页面组件里自己判断 `if locale === "zh-CN"` 来换图。

### 7.2 页面图

扩展 `resolvePageAssets(pageId, audienceContext)`：

```ts
const visualPack =
  audienceContext.marketProfile === "east-asia"
    ? "east-asia"
    : "global-diverse";
```

然后按 pageId 返回：

```ts
case "home":
  return {
    pageId,
    heroImages: {
      light: `/images/markets/${visualPack}/home/hero-after.webp`,
      dark: `/images/markets/${visualPack}/home/hero-after.webp`,
    },
    heroImagesBefore: {
      light: `/images/markets/${visualPack}/home/hero-before.webp`,
      dark: `/images/markets/${visualPack}/home/hero-before.webp`,

    scenarioImages: {
      commute: `/images/markets/${visualPack}/home/scenario-commute.webp`,
```
