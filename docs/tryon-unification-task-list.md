# 统一试妆工作台落地清单

目标：用唯一的 `/tryon` 工作台服务未登录、Free、Pro 和 Premium 用户，套餐差异由权益配置决定，不再复制页面结构、脚本和样式。

## P0：架构与路由

- [x] 盘点三个旧页面的共用结构、功能差异和权限来源
- [x] 以 `src/lib/plans.ts` 作为套餐权益的唯一事实源
- [x] 建立可测试的四类用户体验模型
- [x] 建立唯一 `/tryon` 页面
- [x] 删除 `/tryon-free`、`/tryon-pro`、`/tryon-premium` 旧页面与路由
- [x] 将站内试妆入口统一指向 `/tryon`

## P0：统一体验

- [x] 四类用户共用选择妆容、上传、结果和响应式布局
- [x] 未登录用户可浏览，上传时明确引导登录
- [x] Free 用户显示额度、基础顾问和升级入口
- [x] Pro 用户解锁完整教程、产品参考、高清下载和历史预览
- [x] Premium 用户在相同布局中增加优先队列、长期历史和完整妆容库权益提示
- [x] 后端继续执行额度与功能权限校验，前端显示不作为权限依据

## P1：回归与发布

- [x] 补充体验模型和本地化路由测试
- [x] 检查 375、768、1024、1440 像素布局
- [x] 检查键盘操作、焦点状态和 44 像素触控目标
- [x] 运行类型检查、单元测试和生产构建
- [x] 检查旧链接、深链接参数、登录回跳和支付回跳

## 验收标准

1. 站内不再把用户导航到套餐专属试妆地址。
2. 修改选择栏、结果区或说明栏时只需改一处。
3. 四类身份访问同一 URL，看到与权益一致的状态和操作。
4. 所有入口直接使用 `/tryon`，并保留 `look`、`scenario`、`source` 等参数。
5. Pro/Premium 的教程、高清下载和历史能力没有回退。
