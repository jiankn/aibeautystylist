# AI Beauty Stylist 数据保护影响评估初稿

> 状态：开发启动版初稿，正式上线前需要法律顾问与第三方数据处理方共同复核。  
> 日期：2026-06-07

## 1. 处理目的与边界

产品处理用户主动上传的正脸照片，用于生成非医疗性质的面部与色彩建议、妆效预览和个性化教程。

明确不用于：

- 身份识别、身份验证或执法用途。
- 医疗诊断、健康判断或敏感属性推断。
- 未经用户单独同意的模型训练。
- 未经用户主动分享的公开展示。

## 2. 涉及数据

| 数据 | 敏感度 | 用途 | 保存策略 |
|---|---|---|---|
| 原始自拍 | 高 | 临时 AI 分析与妆效生成 | 处理完成后尽快删除，最长不超过 30 天 |
| 结构化诊断建议 | 中高 | 展示建议和生成教程 | 用户删除或账户删除时删除 |
| 妆效结果图 | 高 | 展示、保存和分享 | 用户删除或账户删除时删除 |
| 订阅与用量 | 中 | 权限、额度和计费 | 按财务与合规要求保存 |
| 行为事件 | 低至中 | 漏斗和质量分析 | 不包含原图或面部特征向量 |

## 3. 数据流

1. 用户浏览妆容并主动选择上传。
2. 上传前展示独立、明确的照片处理同意。
3. 服务端校验图片并存入私有 R2。
4. Worker 将必要数据发送至已审核的 AI 服务商。
5. 结构化诊断和妆效结果返回并保存。
6. 原始自拍按生命周期策略删除。
7. 用户可随时删除上传、结果和诊断数据。

## 4. 法律与合规依据

- 对照片及潜在生物识别数据的处理以用户显式同意为前提。
- 自动化建议必须说明其非专业、非医疗性质，并允许用户自行判断。
- 正式上线前必须确认 Gemini、Evolink、认证、邮件与支付服务的数据处理条款和跨境传输机制。
- Cookie 与非必要分析跟踪在适用地区必须先获得同意。

## 5. 主要风险与缓解措施

| 风险 | 严重度 | 缓解措施 |
|---|---|---|
| 未经明确同意处理正脸照片 | 高 | 上传前独立显式同意；服务端校验同意记录 |
| 原始照片泄露 | 高 | 私有 R2、短期签名访问、最小权限、自动删除 |
| 第三方将数据用于训练 | 高 | 审核 DPA 与服务条款；只选择支持禁训练的数据处理配置 |
| AI 结果错误或带偏见 | 高 | 置信度、免责声明、降级策略、多样化质量测试 |
| 用户无法删除数据 | 高 | 一键删除入口、对象删除重试和删除审计 |
| 权限绕过读取他人结果 | 高 | 每个资源请求进行服务端所有权校验 |
| 支付和身份数据过度集中 | 中 | Stripe 托管支付；产品不保存银行卡数据 |
| 行为分析收集过量 | 中 | 事件白名单；禁止记录自拍和面部特征向量 |

## 6. 数据主体权利

产品必须支持：

- 查看正在保存的数据类别。
- 撤回照片处理同意。
- 删除原图、结果图和诊断记录。
- 删除账户。
- 请求导出结构化诊断和订阅信息。
- 联系人工渠道处理例外情况。

## 7. 开发门禁

- [ ] 上传 API 校验有效显式同意。
- [ ] 私有图片不能通过永久公开 URL 访问。
- [ ] 原图生命周期删除任务可监控和重试。
- [ ] 所有用户资源 API 执行所有权校验。
- [ ] 日志和事件不包含图片、base64、密钥或面部特征向量。
- [ ] 用户删除功能覆盖 D1、R2 和第三方可删除数据。
- [ ] AI 服务商 DPA、训练政策和跨境传输条款完成复核。
- [~] 隐私政策、服务条款、Cookie 同意和 AI 免责声明页面与入口已实现；正式上线前待法律顾问复核。

## 8. 上线前待确认

- 最终数据控制者和数据处理者信息。
- 用户支持与隐私请求联系渠道。
- 各地区的实际开放范围和年龄限制。
- 第三方 AI 服务商的数据驻留、保留和训练政策。
- 原始自拍的最终最长保存时间，目标应短于 30 天。
- 法律顾问对 GDPR、BIPA、CCPA/CPRA 等适用性的复核结论。
