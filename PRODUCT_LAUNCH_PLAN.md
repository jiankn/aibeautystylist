# AI Beauty Stylist 产品落地方案

## 1. 产品判断

产品不是“AI 修图工具”，而是以一次自拍为起点的个性化美妆决策服务。首发版本必须验证用户是否愿意完成：

`发现妆容 → 上传自拍 → 查看诊断/试妆 → 继续尝试或分享 → 付费`

北极星指标为“完成试妆闭环的用户占比”，付费与分享都是闭环后的结果指标。

## 2. 首发范围

### P0：上线必须成立

- 首页：建立价值认知，直接进入试妆或发现妆容。
- 发现妆容：按场景、妆感、经验筛选，并能进入试妆。
- 免费试妆：每天 3 次，明确单次生成规则和分享换次数。
- AI 面容诊断：输出可理解、可执行、非医疗化的面部与色彩建议。
- Pro / Premium 工作台：体现额度、结果管理、高清下载和会员差异。
- 分享卡：传播试妆效果并带回流入口。
- 价格页：清楚解释 Free、Pro、Premium 的价值边界。
- 隐私同意、照片删除、失败重试、额度扣减和订阅状态。

### P1：首发后 2 至 4 周

- 6 至 8 步个性化教程。
- 真实产品 SKU 推荐与 Affiliate 数据埋点。
- 历史记录、收藏妆容、邮件召回。
- 英文版本与 SEO 场景页。

### 暂不做

- 实时 AR、社区、视频教程、海量商城、对话式 AI 顾问。

## 3. 用户状态机

| 状态 | 核心任务 | 页面主行动 |
|---|---|---|
| 未登录访客 | 理解价值、获得灵感 | 开始试妆 |
| Free 用户 | 低成本看到第一次结果 | 上传自拍 |
| 已生成 Free 用户 | 分享或继续试妆 | 分享换次数 / 升级 |
| Pro 用户 | 高频、稳定完成试妆 | 保存、下载、继续试妆 |
| Premium 用户 | 获得更快、更高质量输出 | 管理完整妆容库 |

## 4. 一周落地节奏

1. 第 1 天：冻结页面、状态机、埋点和 API 契约。
2. 第 2 天：完成账户、上传、隐私同意、额度模型。
3. 第 3 天：接入诊断与图片生成，完成失败降级。
4. 第 4 天：完成 Free、Pro、Premium 工作台和分享卡。
5. 第 5 天：完成 Stripe、价格页、订阅回调和权限校验。
6. 第 6 天：全链路 QA、移动端、无障碍和性能优化。
7. 第 7 天：灰度发布，监控闭环完成率、成本和失败率。

## 5. 数据与验收

- `home_start_tryon_click`
- `discover_filter_apply`
- `look_selected`
- `photo_consent_accepted`
- `photo_upload_success`
- `diagnosis_generated`
- `tryon_generated`
- `share_card_created`
- `share_completed`
- `pricing_viewed`
- `checkout_started`
- `subscription_activated`

上线门槛：

- 上传至结果成功率不低于 95%。
- P95 首个结果等待时间控制在 45 秒内，并有明确进度反馈。
- 额度扣减具备幂等性，失败任务自动返还额度。
- 用户能删除原图、结果图和诊断记录。
- 所有页面支持 375px、768px、1440px 三档尺寸，无横向溢出。
- 关键按钮可键盘操作，文字与背景满足 WCAG AA。

## 6. UI / UX 执行原则

- `UI` 目录 8 张参考图是页面视觉基准，不能自行改成另一套风格。
- 导航、品牌、按钮、卡片圆角、色彩和会员状态使用统一设计令牌。
- 每个页面只有一个主行动，次要操作降低视觉权重。
- Free 页面必须解释额度规则；付费页面只展示当前计划和管理动作，不重复售卖。
- AI 诊断使用“建议、参考、置信度”表达，避免绝对判断。
- 上传前展示隐私承诺，生成失败给出原因、重试和额度返还说明。
